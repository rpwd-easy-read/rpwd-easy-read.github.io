# Full site audit, fresh-eyes verification: 2026-07-13

Auditor session per
`equitabledocs-program\prompts\fable-rpwd-full-audit.md`. The auditor
did not build this code. Everything claimed live was verified
programmatically, not trusted. Method, results, findings, and the
fixed-versus-flagged split below.

## Method

- **Live parity first.** All seven served files (index.html, js/app.js,
  css/main.css, content.json, sw.js, manifest.webmanifest,
  content/training/ch-02.json) fetched from
  https://rpwd-easy-read.github.io/ and SHA-256 compared against
  `git show HEAD:<file>`. All seven matched, and main == origin/main,
  so local headless sweeps are valid evidence for the live site.
- **Harness.** puppeteer-core driving installed Chrome (headless),
  axe-core 4.12 injected from node_modules, local static server on the
  repo root. Scripts and raw JSON results in
  `design-review\audit-2026-07-13\`.
- **Reflow** via the same-origin iframe probe (headless window clamps
  at 500px on this machine, per the calm-verify record).
- **Lighthouse** run against the live origin on six route classes.
- **TTS** verified against a recording stub installed with
  `Object.defineProperty` before app boot (plain assignment to
  `window.speechSynthesis` silently fails; the first harness pass had
  that bug and its TTS failures were harness artifacts, not app bugs).

## Per-sweep results

### 1. axe on every route: PASS after fixes (zero, 257 routes)

Home, map, 5 band pages, 17 chapters, all 102 sections, all 102 /more
panels, training landing, all 22 module slide routes, about,
using-the-act, help, complain, your-rights, and a nonsense route.
**Zero axe violations on all 257 routes**, both before and after the
in-session fixes.

### 2. Content integrity per section: PASS (all 102)

Per section: `plain_title` present, `official_text` non-empty, Section
badge renders "Section N", illustration loads with naturalWidth > 0,
alt text non-empty and never the generic fallback, no `undefined`,
`[object Object]`, `NaN`, or `${` leaking into DOM text on any route.
All clean. Chapter pages render every plain title exactly as authored.

### 3. Easy Read text untouched: PASS (proved)

`content.json` at design-v1 versus HEAD, restricted to `plain_text`
and `use_when` across all 102 sections: **zero changes**. The
`title` field is also byte-identical to design-v1.

### 4. Bullets fidelity: PASS

Exactly S3 to S15 carry `know_more_bullets` in data and render the
"More in plain words" layer; all other 89 sections show only the
actual-words layer. No empty bullet lists, no stray kickers anywhere
(verified in the rendered DOM on all 102 /more panels).

### 5. Interaction walks

- Know more panel: Enter opens onto the sub-route, focus lands on the
  panel heading, aria-expanded tracks, Escape closes and returns focus
  to the trigger. PASS.
- Slide deck: ArrowRight walks slides 1 to 22 with the counter correct
  at every step, ArrowLeft walks back, details reveal toggles, focus
  lands on the slide heading per render, mark-complete stores and
  clears, last slide offers Finish back to Training. PASS.
- Search (home and map): hit, no-hit message, clear-empties. PASS.
- Big text: toggles with aria-pressed, persists across reload. PASS.
- TTS state machine: cancel-then-deferred-speak order, Stop button
  swap on start, Stop cancels, route change cancels, panel-open scope
  includes bullets plus verbatim, panel-close cancels, panel-closed
  scope is plain only. PASS.
- **FAIL found and fixed: skip link 404 (P1, F1 below).**
- **FAIL found and fixed: stale Escape listener rewrites URL (P2, F2).**

### 6. Links and assets: PASS

Every internal href on every rendered route resolves to a real route
(the crawl surfaced the `#fn:N` fragment links, see F1); every img on
every route loads; no mixed content; **console 100 percent clean**
across all 257 routes: no errors, no warnings, no failed requests, no
4xx/5xx.

### 8. Reflow at 320px: PASS after one fix

Manifest parses (2 icons, both load as image/png). SW registers and

### 10. Copy sweep: PASS with flags

No em or en dashes in any rendered UI string (the `title` data field
contains em dashes but is never rendered; it feeds search matching
only, and predates design-v1 unchanged). No ableist vocabulary in any
UI string, bullet, or training slide. No lorem or placeholder text
(the `[xxxx]` in the ch-02 worked example is a deliberate redaction in
a complaint-letter template). Footer is exactly two lines with the
copyright line verbatim. About carries the credits (author, CPACC,
repository link). One antithesis construction flagged, not fixed (F6).
